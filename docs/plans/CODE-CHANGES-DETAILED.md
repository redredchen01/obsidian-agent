### Add After Section "Architecture Overview" or "Index Management"

```markdown
## Knowledge Graph Format (_graph.md)

The knowledge graph table generated by `IndexManager.rebuildGraph()` uses a 4-column format:

| Column | Purpose | Values | Notes |
|--------|---------|--------|-------|
| Source | Note initiating the relationship | `[[filename]]` | Obsidian link syntax |
| Links To | Target note | `[[filename]]` | Obsidian link syntax |
| Type | Relationship category | `strong`, `medium`, `weak`, `nav` | Based on tag overlap for regular links; `nav` for temporal links |
| Navigation | Directional metadata | `nav-prev`, `nav-next`, empty | Journal entries only; empty for regular relationships |

### Example Rows

```
| [[2026-03-27]] | [[2026-03-28]] | nav | nav-next |
| [[2026-03-27]] | [[2026-03-26]] | nav | nav-prev |
| [[project-a]]  | [[project-b]]  | strong | |
| [[resource-1]] | [[resource-2]] | medium | |
```

### Navigation Tags

**nav-prev**: Points to the previous journal entry (chronologically backward)
- Used when linking: `[[2026-03-27]] -> [[2026-03-26]]`
- Enables backward navigation through daily notes

**nav-next**: Points to the next journal entry (chronologically forward)
- Used when linking: `[[2026-03-27]] -> [[2026-03-28]]`
- Enables forward navigation through daily notes

### Implementation Details

Journal entries (files in `journal/` directory with `YYYY-MM-DD` names) automatically generate temporal links:
1. Link to previous day (if exists): `nav-prev` tag
2. Link to next day (if exists): `nav-next` tag

This creates a bidirectional timeline chain that enables:
- Timeline reconstruction
- Chronological navigation in graph tools
- Filtering by navigation type
- Integration with Obsidian timeline plugins

### Design Rationale

The 4-column format with semantic nav tags provides:
- **Clarity**: Direction explicitly encoded (not inferred)
- **Consistency**: All rows have same column count
- **Extensibility**: Pattern scales to nav-parent, nav-child, nav-sequence, etc.
- **Tooling**: Graph analysis tools can filter by Type or Navigation column
```

**Purpose**:
- Documents the expected format
- Explains semantic meaning of nav-prev/nav-next
- Provides examples
- Helps future maintainers understand design decisions
- Enables graph tools to interpret navigation properly
