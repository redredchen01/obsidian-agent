```markdown
## Knowledge Graph Format (_graph.md)

The knowledge graph table generated by `IndexManager.rebuildGraph()` uses a 4-column format:

| Column | Purpose | Values |
|--------|---------|--------|
| Source | Note initiating relationship | `[[filename]]` |
| Links To | Target note | `[[filename]]` |
| Type | Relationship category | `strong`, `medium`, `weak`, `nav` |
| Navigation | Directional metadata | `nav-prev`, `nav-next`, empty |

### Navigation Tags

**nav-prev**: Points to previous journal entry (backward in time)
**nav-next**: Points to next journal entry (forward in time)

### Design Rationale

- Clarity: Direction explicitly encoded
- Consistency: All rows have same column count
- Extensibility: Pattern scales to nav-parent, nav-child, etc.
- Tooling: Enables graph analysis and filtering
```
